## Export Configuration

### Format Settings

```yaml
reporting:
  exports:
    # Default format
    default_format: "pdf"

    # Available formats
    formats:
      pdf:
        enabled: true
        template: "professional"
        include_charts: true
        page_size: "letter"  # letter, A4

      html:
        enabled: true
        interactive: true
        include_scripts: true

      markdown:
        enabled: true
        include_diagrams: true
        github_flavored: true

      csv:
        enabled: true
        delimiter: ","
        include_headers: true

      json:
        enabled: true
        pretty_print: true

      excel:
        enabled: true
        include_charts: true
        multiple_sheets: true
```

### Output Settings

```yaml
reporting:
  exports:
    output:
      # Directory
      directory: ".proagents/reports/exports/"

      # Naming
      filename_format: "{{report_type}}-{{date}}.{{format}}"

      # Compression
      compress: true
      compression_format: "zip"  # zip, tar.gz

      # Cleanup
      retention_days: 90
```

## Export Commands

### CLI Export

```bash
# Export velocity report as PDF
proagents report velocity --export pdf

# Export to specific file
proagents report quality --export pdf --output ./reports/q1-quality.pdf

# Export multiple formats
proagents report velocity --export pdf,csv,json

# Export all reports
proagents report export-all --format pdf

# Export with date range
proagents report velocity --from 2024-01-01 --to 2024-01-31 --export pdf
```

### Batch Export

```bash
# Export all reports for a period
proagents report batch-export \
  --from 2024-01-01 \
  --to 2024-01-31 \
  --formats pdf,csv \
  --output ./monthly-reports/

# Schedule recurring export
proagents report schedule \
  --report velocity \
  --format pdf \
  --frequency weekly \
  --email team@company.com
```

## CSV/Excel Exports

### CSV Configuration

```yaml
reporting:
  exports:
    csv:
      # Format
      delimiter: ","
      quote_char: "\""
      escape_char: "\\"
      line_ending: "\n"  # \n, \r\n

      # Headers
      include_headers: true
      header_row: true

      # Data handling
      null_value: ""
      date_format: "YYYY-MM-DD"
      number_format: "0.00"
```

### Excel Configuration

```yaml
reporting:
  exports:
    excel:
      # Multiple sheets
      multiple_sheets: true

      sheets:
        - name: "Summary"
          data: "summary"
        - name: "Velocity"
          data: "velocity_details"
        - name: "Quality"
          data: "quality_metrics"
        - name: "Raw Data"
          data: "raw_metrics"

      # Formatting
      formatting:
        header_style:
          bold: true
          background: "#1a73e8"
          color: "#ffffff"

        # Conditional formatting
        conditional:
          - column: "coverage"
            rule: "< 80"
            style:
              background: "#ffcccc"
          - column: "coverage"
            rule: ">= 80"
            style:
              background: "#ccffcc"

      # Charts in Excel
      include_charts: true
```

```yaml

  exports:
```

```yaml
      # Daily summary
      - name: "daily_summary"

        format: "html"
        frequency: "daily"
        time: "08:00"
        timezone: "America/New_York"

          - "team@company.com"

      # Weekly report
      - name: "weekly_report"
        report: "all"
        format: "pdf"
        frequency: "weekly"
        day: "monday"
        time: "09:00"

          - "management@company.com"
```

## Best Practices

1. **Choose Right Format**: PDF for sharing, CSV for analysis
2. **Schedule Regular Exports**: Automate recurring reports
3. **Archive Important Reports**: Keep historical data
4. **Secure Sensitive Data**: Encrypt exports with sensitive info
5. **Test Templates**: Verify formatting before distribution
6. **Monitor Storage**: Clean up old exports regularly
