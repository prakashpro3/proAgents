# Code Review Prompts

Universal prompts for comprehensive code review phase.

---

## Quick Start

```
/review
```

---

## Review Types

### Automated Review

```
Run automated code review for: [FEATURE/FILES]

Checks to run:
1. Linting (ESLint/Prettier)
2. Type checking (TypeScript)
3. Static analysis
4. Security scanning
5. Bundle size analysis
6. Test coverage check
```

### AI-Assisted Review

```
Perform AI code review for: [FEATURE]

Review focus areas:
1. Code correctness
2. Logic errors
3. Security vulnerabilities
4. Performance issues

```

### Peer Review Request

```
Create peer review request for: [FEATURE]

Include:
- Summary of changes
- Files modified
- Testing done
- Areas needing attention
- Related issues/tickets
```

## Security Review

### Security-Focused Review

```
Perform security review for: [CODE]

CHECK FOR OWASP TOP 10:

1. Injection:
□ SQL queries parameterized
□ Command injection prevented
□ LDAP injection prevented

2. Broken Authentication:
□ Strong password requirements
□ Session management secure
□ MFA supported (if needed)

3. Sensitive Data Exposure:
□ Data encrypted in transit
□ Data encrypted at rest
□ No sensitive data in logs

4. XML External Entities:
□ XXE prevented
□ Parser configured securely

5. Broken Access Control:
□ Authorization enforced
□ CORS configured properly
□ Direct object references secured

6. Security Misconfiguration:
□ Default credentials removed
□ Error handling doesn't leak info
□ Security headers present

7. Cross-Site Scripting (XSS):
□ Output encoded
□ CSP configured
□ User input sanitized

8. Insecure Deserialization:
□ Deserialization validated
□ Types checked

9. Known Vulnerabilities:
□ Dependencies up to date
□ No known vulnerable packages

10. Insufficient Logging:
□ Security events logged
□ Logs don't contain sensitive data
```

### Secrets Check

```
Check for hardcoded secrets in: [CODE]

Look for:
□ API keys
□ Passwords
□ Private keys
□ Access tokens
□ Connection strings
□ AWS credentials
□ Firebase configs
□ OAuth secrets

Found secrets should:
- Move to environment variables
- Use secrets manager
- Never commit to git
```

## Review Feedback

### Provide Review Feedback

```
Provide review feedback for: [CODE]

Format feedback as:

MUST FIX (Blocking):
1. [Issue] - [Location] - [Why it's critical] - [Suggested fix]

SHOULD FIX (Non-blocking but important):
1. [Issue] - [Location] - [Why] - [Suggested fix]

CONSIDER (Suggestions):
1. [Suggestion] - [Why it might help]

POSITIVE:
1. [What's done well]
```
